Chicago Taxi Fare Prediction¶

Taxi 데이터를 활용하여 주행거리(TRIP_MILES)와 주행시간(TRIP_SECONDS)에 따른 요금(FARE)을 선형회귀로 예측하고 성능을 비교한다.

분석 목표¶

  1. TRIP_MILES와 FARE 사이의 관계를 산점도로 확인한다.
  2. TRIP_SECONDS와 FARE 사이의 관계를 산점도로 확인한다.
  3. TRIP_MILES만 사용한 단순 선형회귀 모델을 학습한다.
  4. TRIP_SECONDS만 사용한 단순 선형회귀 모델을 학습한다.
  5. TRIP_MILES와 TRIP_SECONDS를 동시에 사용한 다중 선형회귀 모델을 학습한다.
  6. 각 모델의 가중치(weight), 편향(bias), RMSE를 비교한다.
  7. Correlation Matrix와 VIF를 통해 다중공선성 가능성을 해석한다.

# 7. VIF 계산
# =========================
# VIF = 1 / (1 - R^2)
# 독립변수 간 다중공선성 정도를 확인하기 위해 계산한다.
# 일반적으로 VIF가 5 이상이면 주의, 10 이상이면 심각한 다중공선성으로 본다.

def calculate_vif(df_features):
    vif_result = []

    for target_col in df_features.columns:
        other_cols = [col for col in df_features.columns if col != target_col]

        model = LinearRegression()
        model.fit(df_features[other_cols], df_features[target_col])

        r_squared = model.score(df_features[other_cols], df_features[target_col])
        vif = 1 / (1 - r_squared)

        vif_result.append({
            "Feature": target_col,
            "R_squared_with_other_features": r_squared,
            "VIF": vif
        })

    return pd.DataFrame(vif_result)

vif_df = calculate_vif(taxi[["TRIP_MILES", "TRIP_SECONDS"]])

def train_and_evaluate(model_name, X_train, X_test, y_train, y_test):
    model = LinearRegression()
    model.fit(X_train, y_train)

    y_pred = model.predict(X_test)
    rmse = np.sqrt(mean_squared_error(y_test, y_pred))

    print("=" * 70)
    print(model_name)
    print("Features:", list(X_train.columns))
    print("Weights:", model.coef_)
    print("Bias:", model.intercept_)
    print("RMSE:", rmse)

    return model, y_pred, rmse

model_miles, pred_miles, rmse_miles = train_and_evaluate(
    "Simple Linear Regression: TRIP_MILES -> FARE",
    X_miles_train, X_miles_test, y_train, y_test
)

model_seconds, pred_seconds, rmse_seconds = train_and_evaluate(
    "Simple Linear Regression: TRIP_SECONDS -> FARE",
    X_seconds_train, X_seconds_test, y_train, y_test
)

model_multi, pred_multi, rmse_multi = train_and_evaluate(
    "Multiple Linear Regression: TRIP_MILES + TRIP_SECONDS -> FARE",
    X_multi_train, X_multi_test, y_train, y_test
)

결과 해석¶

1. 산점도 해석¶

TRIP_MILES와 FARE의 산점도에서는 주행거리가 증가할수록 요금도 함께 증가하는 뚜렷한 양의 선형관계가 나타났다. 점들이 회귀선 주변에 비교적 밀집되어 있으므로, 주행거리는 택시 요금을 설명하는 핵심 변수라고 볼 수 있다.

TRIP_SECONDS와 FARE 역시 양의 관계를 보였지만, 같은 주행시간에서도 요금이 넓게 퍼져 있었다. 따라서 주행시간만으로 요금을 예측하는 모델은 주행거리 기반 모델보다 예측 오차가 클 가능성이 높다.

2. 회귀계수와 편향 해석¶

TRIP_MILES만 사용한 단순 선형회귀식은 다음과 같다.

FARE = 2.2751 × TRIP_MILES + 5.0386

이는 주행거리가 1마일 증가할 때 요금이 평균적으로 약 2.2751 증가한다는 의미이다. 편향 5.0386은 기본요금에 가까운 값으로 해석할 수 있다.

TRIP_SECONDS만 사용한 단순 선형회귀식은 다음과 같다.

FARE = 0.0151 × TRIP_SECONDS + 3.9493

즉, 주행시간이 1초 증가할 때 요금은 평균적으로 약 0.0151 증가한다. 초 단위 변수이기 때문에 계수 값은 작게 나타난다.

두 변수를 함께 사용한 다중 선형회귀식은 다음과 같다.

FARE = 2.0163 × TRIP_MILES + 0.0025 × TRIP_SECONDS + 3.8571

다중회귀에서는 다른 변수를 통제한 상태에서 각 변수의 영향을 해석한다. 따라서 TRIP_MILES의 계수 2.0163은 주행시간이 동일할 때 주행거리 1마일 증가에 따른 요금 증가량이고, TRIP_SECONDS의 계수 0.0025는 주행거리가 동일할 때 주행시간 1초 증가에 따른 요금 증가량이다.

3. Correlation Matrix와 다중공선성 해석¶

Correlation Matrix에서 TRIP_MILES와 FARE의 상관계수는 약 0.975로 매우 높았다. 반면 TRIP_SECONDS와 FARE의 상관계수는 약 0.830으로, 양의 상관관계는 있지만 주행거리보다 낮았다. 이를 통해 요금 예측에는 주행시간보다 주행거리가 더 직접적인 영향을 주는 변수임을 확인할 수 있다.

또한 TRIP_MILES와 TRIP_SECONDS 사이의 상관계수는 약 0.801로 비교적 높게 나타났다. 즉, 두 독립변수는 서로 관련된 정보를 일부 공유하므로 다중공선성 가능성이 있다. 그러나 VIF 값은 약 2.79로, 일반적으로 심각한 다중공선성 기준인 5 또는 10보다 낮다. 따라서 심각한 다중공선성 문제는 아니지만, 두 변수가 유사한 정보를 일부 설명하기 때문에 회귀계수에는 영향을 줄 수 있다.

실제로 TRIP_MILES만 사용했을 때 계수는 2.2751이었지만, TRIP_SECONDS를 함께 사용한 다중회귀에서는 2.0163으로 감소하였다. 이는 단순회귀에서 주행거리가 단독으로 설명하던 요금 변동의 일부를 주행시간 변수가 함께 설명했기 때문이다.

4. RMSE 비교 해석¶

RMSE는 예측값과 실제값 사이의 평균적인 오차 크기를 의미하며, 값이 작을수록 예측 성능이 좋다.

세 모델의 RMSE는 다음과 같다.

모델 RMSE
TRIP_MILES only 3.8508
TRIP_SECONDS only 9.6552
TRIP_MILES + TRIP_SECONDS 3.6046

TRIP_SECONDS만 사용한 모델은 RMSE가 9.6552로 가장 컸고, TRIP_MILES만 사용한 모델은 RMSE가 3.8508로 훨씬 낮았다. 따라서 주행시간보다 주행거리가 요금 예측에 더 적합한 변수임을 알 수 있다.

가장 좋은 성능은 TRIP_MILES와 TRIP_SECONDS를 함께 사용한 다중 선형회귀 모델에서 나타났으며, RMSE는 3.6046이었다. 다만 TRIP_MILES 단독 모델과의 차이가 크지는 않기 때문에, 이 데이터에서는 주행거리가 요금 예측의 대부분을 설명하고, 주행시간은 추가적인 보조 정보를 제공한다고 해석할 수 있다.
